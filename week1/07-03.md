### 2. 오늘 가장 의미있었던 기술적 판단
- 내가 내린 판단:
  - RequestHandler가 요청 전달, 응답 전달 같이 가벼운 책임만 지니도록 분리한다.
- 그렇게 판단한 이유:
  - 이번 스텝 3를 구현 이전에는 RequestHandler가 request 객체 생성, 로깅, 응답 생성 등 여러가지 기능을 하고 있었다.
  - 이는 아직 분리할 정도로 기능이 많지 않다 생각하였기 때문인데, 이런 상황에서 API 호출을 도입해야했다.
  - 그렇기에 관련 로직을 RequestHandler에서 분리하여 API 처리를 담당할 processor에 넘기고, requestHandler는 요청 전달, 응답 전달 과 같이 가벼운 책임만 지니도록 하는게 더 바람직하다 판단했다.
- 고려했지만 선택하지 않은 대안:
  - API가 이제 1개 추가 된 것이기에 API 처리 로직을 기존처럼 RequestHandler에 추가하는 방안도 고려했다.
    구현은 간단하지만, 요청 처리 방식이 늘어날수록 RequestHandler의 책임과 변경 이유가 함께 증가하므로 선택하지 않았다.
- 이 판단이 코드 품질/테스트/유지보수에 미친 영향:
  - RequestHandler는 요청과 응답을 전달하는 역할에 집중하게 되어 책임이 명확해졌다.
  - API 처리 로직을 processor 단위로 독립적으로 테스트할 수 있게 되었다.
  - 새로운 API 처리 방식이 추가되더라도 RequestHandler의 변경을 최소화할 수 있게 되었다.
  - 다만 processor라는 구조가 추가되면서 호출 흐름을 한 단계 더 따라가야 하는 비용이 생겼다.

### 6. 내가 최종 선택한 가장 잘 작성했다고 생각하는 프롬프트
원본 프롬프트:
> 
> 가상 스레드가 일반 스레드 위에서 올라가는 경량 스레드인 만큼 항상 스레드 보다 나은 선택인가요?
> 저의 생각으로는 가상 스레드에 pinning이 생겨도 일반 플랫폼 스레드와 동일하기에 항상 일반 스레드보다 가상 스레드가 이점이 있다 생각합니다.
> 사례와 근거를 들어 이야기해주세요.

선택한 이유:
이번 주간 과제에서는 주로 AI를 구현보단 검증 및 질문, 토론으로 많이 사용하였다.
그러한 프롬프트 중 이 프롬프트가 가장 나의 궁금증을 잘 해소해주었다 생각하여 선택했다.

### 7. 원본 프롬프트 구조 분석
- 문제 상황: 가상 스레드가 플랫폼 스레드보다 항상 유리한지, 특히 pinning이 발생하는 상황에서도 이점이 유지되는지 궁금해한다.
- 원하는 결과를 어떻게 요구했는가: 사례와 근거를 들어 생각이 타당한지 이야기해달라 하였다.
- 필요한 맥락이 포함되어 있는가: 자신의 생각이 맞는지 검증하려는 기본 맥락을 있다 판단한다.
- 판단 기준이나 제약 조건이 있는가: 구체적으로 제시되지 않았다.
- 출력 방식이 명확한가: 구체적으로 제시되지 않았다.
- 나의 가설, 우려, 의도가 드러나는가: "pinning이 발생해도 플랫폼 스레드와 같아질 뿐이므로 가상 스레드가 항상 더 낫다"는 가설이 드러난다.

### 8. AI에게 프롬프트를 개선 요청하기
- 프롬프트 개선안
```
Java의 가상 스레드가 플랫폼 스레드보다 항상 더 나은 선택인지 판단하고 싶습니다.
저는 가상 스레드가 플랫폼 스레드 위에서 실행되는 경량 스레드이므로, pinning이 발생하더라도 최악의 경우 플랫폼 스레드와 비슷해질 뿐이며 전체적으로는 가상 스레드가 더 유리하다고 생각합니다.
다음 기준으로 제 가설을 검토해 주세요.
- I/O 중심 작업과 CPU 중심 작업에서의 차이
- pinning이 처리량과 플랫폼 스레드 사용량에 미치는 영향
- 가상 스레드의 생성·스케줄링 비용
- ThreadLocal, 메모리 사용량, 디버깅 및 모니터링 측면
- 가상 스레드가 유리한 사례와 플랫폼 스레드가 더 적절한 사례
각 항목에 대해 동작 원리와 구체적인 사례를 제시하고, 제 가설에서 맞는 부분과 잘못된 부분을 구분해 주세요. 마지막에는 “가상 스레드를 기본 선택으로 사용해도 되는 조건”과 “플랫폼 스레드를 선택해야 하는 조건”을 표로 정리해 주세요.
```

```
- 답변이 사용자의 주장에 치우치는 문제를 예방할 수 있다. 또한 구체적인 기준점을 제시하여 어느 방식이 어느 경우에 더 좋은지 확인 할 수 있다.
```
